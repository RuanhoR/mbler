# v0.2.4 (2026-05-02)

### Highlights

This release marks a major architectural overhaul with a complete rewrite of the CLI argument parsing, introduction of build caching, and expanded project management capabilities.

### Features

- **Major refactor**: Complete rewrite of command-line argument parsing and project structure
- **Build cache**: Implemented `BuildCacheManager` with memory/file/auto modes, integrated with Rollup caching
- **Configuration hooks**: Extended `mbler.config.js` with `onStart`, `onEnd`, `onWarn` lifecycle hooks and custom Rollup plugin support
- **`defineConfig`**: Added `defineConfig()` helper for type-safe configuration
- **Script configuration**: Support for JavaScript/TypeScript config files (`mbler.config.ts`)
- **Console IO cache**: Caching for console input/output operations
- **File output name replacement**: Watcher mode now supports dynamic output filename replacement
- **New CLI commands**: `config` (configuration management), `profile` (user information), `view` (package info)
- **MCX native handling**: When `lang = mcx`, Rollup TS extension is disabled in favor of internal MCX processing for better type support
- **i18n improvements**: Enhanced English and Chinese internationalization
- **MCX TSC check**: Added TypeScript type checking for MCX scripts via `mcx-tsc`
- **Project structure**: New `example/` directory replacing `test/` for better semantics
- **Template system**: Enhanced `init` command with template directory handling, welcome prompt, and new template files
- **Version control**: New `version` command for version management
- **Utility functions**: Added player/block data handling utilities
- **SAPI auto-download**: Automatic SAPI version loading from cache or network
- **Manifest generation**: Complete manifest generation for behavior/resource packs
- **MNX marketplace publish**: WIP support for publishing to MNX marketplace (`pmnx publish`)
- **Improved watcher**: Enhanced watch mode logging

### Fixes

- Fixed ESM import errors in various modules
- Fixed URL encoding and token validation fallback logic
- Fixed Minecraft API dependency version generation
- Fixed manifest generation and build bundle option interaction
- Fixed `@mbler/mcx-core` dependency resolution and template setup
- Fixed `tslib` not found when installed globally via `npm i -g mbler`
- Fixed TTY detection in non-TTY environments (commander `setRawMode`)
- Fixed template resolution during `mbler init`
- Fixed i18n initialization and build check interaction

### Chores

- Upgraded `@mbler/mcx-core` to `0.0.3-beta` → `0.0.4-beta.r1`
- Repackaged project dependencies; moved Rollup to devDependencies
- Added pnpm workspace support
- Added `project call` capability to MCX plugin interface
