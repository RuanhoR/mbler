# v0.1.3 (2026-02-09)

### Highlights

- **MCX module extraction**: The `mcx` module was moved to a separate repository (`@mbler/mcx-core`) and is now consumed as an npm dependency. This reduces mbler's codebase size significantly and enables independent versioning of the MCX compiler.

### Features

- **Init enhancement**: Auto-adds `@mbler/mcx` dependency when MCX language is selected during `mbler init`
- **Config-driven output**: `dev.ts` now reads `mbler.config.json` output directory and adds it to ignore list
- **Type declarations**: TypeScript declaration generation enabled in build config

### Fixes

- Fixed MCX import error
- Fixed `dev.ts` not respecting config's output directory
- Fixed `mcx use` error

### Chores

- Updated package versions
- Standardized naming convention (e.g. `getModule.ts` → `module.ts`)
